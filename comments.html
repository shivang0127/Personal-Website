2. CSS Breakdown

This part aims at breaking down all the Elements and Classes styled within the CSS code to modify the design and looks of the webpages.

Name Type Purpose
* Universal Selector Applies consistent sizing to all the elements.
body Element Works on font, text color, backrgound color, and all the other properties of the chief body.
main Element Sizes and styles the main content area between the header and the footer.

.card Class This class simply creates a white box with padding and rounded corners for grouped content.

.heading-3 Class Similar to the .heading-1 class, this is created to style the h3 tag.
.home-img img Class & Element Limits the size of the image to fit on the page.
.footer Class Styles the footer containing additional accessibility links to other pages as well as meta-info.
.footer-notes Class Aligns the notes in the footer to the left, along with adding adequate margin.
.footer-links a Class + Element Styles all the anchor links within the anchor tag by changin their color and changing the default underline property disappear.

.flowchart Class Creates a flexbox layout for the flowchart within the past page.
.flochart-box Class Customizes the content (especially the text) within each box of the flowchart.
.subject-box Class Changes the boxes within which the names of the subjects are such that the boundary is dashed.
.arrow Class Changes the size and color of the arrow.
.muted Class The text within this class appears muted grey. Used for secondary text.
.table Class Collapses the borders of the tables so that no gap appears between the cells, and also formats the table to full width.

.table th Class + Element Used to differentiate the table header by making it have distinct color.

.table-term-headings Class Styles the term headings with lavendar backgrounds.

3. Design & Aesthetics

  • Initially, in my mind I had a Theme inspired by Minecraft which would have the header and footer as obsidian blocks and the central background as the nether portal texture.
  • However, I realized (after raising a question in the discussion board) that using such a theme can lead to copyright infringement, due to which I scraped that idea.
  • As I had already coded quite a large part of the website before making that change, I decided to keep the colors and just remove the copyright-ed images.
  • Color scheme:
    Purple buttons,
    White cards/text,
    Dark contrasts.
  • Consistent typography: used the Arial font style across all the pages to make the text appear consistent and only modified the text in certain areas with changed color or font-size instead of changing the entire font-style.
  • Used larger font sizes for headings to ensure hierarchy.
  • Hover glow effects improve interactivity without heavy animation.

4. Accessibility & Compatibility

  • Works with CSS disabled (content still visible and navigable).
  • The image includes a descriptive alt text.
  • Font sizes and line spacing are readable.
  • Responsive design tested on smaller screens.

5. Evaluation of the Site

This is the first website I have ever created, yet I am somewhat satisfied with the result as I spent countless hours on it and it didn't turn out so bad afterall! The site succeeds in being visually engaging while maintaining clarity and usability. Its strongest aspects include the consistent layout, thematic design, and responsiveness. I have tried using as many different CSS features as I could find on the internet that were relevant to this assignment, such as tables, different effects on links, proper alignment of divs and text, links to other webpages, flowcharts, cards, etc which not only helped me make this website look and flow better, but also improved my web development skills a ton.

Possible area of improvement in my view could be less text, as I belive I did write a lot in certain pages which was not needed... Other than that I believe the website fulfils all the criteria and satisfies the specification well.
